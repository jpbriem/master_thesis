```python
def longest_palindromic_subsequence(input):
    n = len(input)
    dp = [[0] * (n + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(i):
            if input[i - 1] == input[j]:
                dp[i][j] = dp[i - 1][j] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[-1][-1]
```

Explanation:

The problem statement asks to find the length of the longest palindromic subsequence in a given string. A palindrome is a sequence that reads the same backward as forward.

We can solve this problem using dynamic programming. We create an array `dp` where `dp[i][j]` represents the length of the longest palindromic subsequence ending at index `i` and starting at index `j`.

To compute `dp[i][j]`, we consider two cases:

- If `input[i - 1] == input[j]`, then we have found a new character that matches the previous one, so we add 1 to the length of the longest palindromic subsequence ending at index `i - 1` and starting at index `j - 1`. This gives us `dp[i][j] = dp[i - 1][j] + 1`.
- Otherwise, there are two possibilities:

  - The current characters do not match, so we take the maximum of the lengths of the longest palindromic subsequence ending at index `i - 1` and starting at index `j - 1` and the length of the longest palindromic subsequence ending at index `i` and starting at index `j`. This gives us `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])`.

Finally, the length of the longest palindromic subsequence in the entire string is stored in `dp[-1][-1]`.

Note that this solution has a time complexity of O(n^2), where n is the length of the input string.
