```python
def longest_common_subsequence(X, Y):
    m = len(X)
    n = len(Y)

    dp = [[0 for _ in range(n+1)] for _ in range(m+1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    result = []
    index = dp[m][n]
    while index > 0:
        result.append([X[m - index], Y[n - index]])
        index -= 1

    return result[::-1]
```

Explanation:

This function uses dynamic programming to solve the problem. The idea is to build a table `dp` where `dp[i][j]` represents the length of the longest common subsequence between the first `i` characters of string `X` and the first `j` characters of string `Y`. We start by initializing all elements of `dp` to zero.

Then we iterate over each character of both strings, and if they are equal, we add one to the value at `dp[i][j-1]`, since the current character can be included in the LCS. Otherwise, we take the maximum of the values at `dp[i-1][j]` and `dp[i][j-1]`.

Finally, we backtrack through the table `dp` to construct the LCS itself. Starting from the bottom-right corner of the table, we move diagonally up-left until we reach an element that has not been assigned yet. At each step, we append the two characters to our result list and update their indices in `dp` accordingly.

The time complexity of this solution is O(m * n), where m and n are the lengths of the input strings `X` and `Y`. This is because we need to fill out the entire table `dp` with m + n rows and columns. The space complexity is also O(m * n) due to the same reason.
