```python
class Solution:
    def groupAnagrams(self, strs):
        """
        :type strs: List[str]
        :rtype: List[List[str]]
        """
        result = []
        
        if len(strs) == 0:
            return result
        
        seen = {}
        for s in strs:
            key = "".join(sorted(s))
            if key not in seen:
                seen[key] = [s]
            else:
                seen[key].append(s)
                
        for k, v in seen.items():
            result.append(v)
            
        return result
```

Explanation:

- We initialize an empty list `result` to store the groups of anagrams.
- We check if the input list is empty. If it is, we return an empty list as there are no anagrams to group.
- We create a dictionary called `seen` to keep track of which strings have already been encountered. Each string will be its own key, and its value will be a list containing all the strings that have the same characters sorted in the same order.
- For each string `s` in the input list, we sort its characters in alphabetical order using the `sorted()` function. Then we take the resulting string as our key and append it to the `seen` dictionary if it doesn't exist yet, or add it to the existing list if it does.
- Finally, we iterate over the keys in the `seen` dictionary and append each list of strings to the `result` list.

This solution has a time complexity of O(n*m), where n is the length of the longest string in the input list and m is the number of unique characters in the longest string. This is because we need to sort each string before adding it to the dictionary, and sorting takes O(m log m) time. The space complexity is also O(n*m), since we need to store the groups of anagrams in memory.
