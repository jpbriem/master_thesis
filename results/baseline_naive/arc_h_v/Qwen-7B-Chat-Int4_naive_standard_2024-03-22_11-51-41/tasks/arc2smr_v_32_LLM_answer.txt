```python
class Solution:
    def groupAnagrams(self, strs):
        """
        :type strs: List[str]
        :rtype: List[List[str]]
        """
        result = []
        
        # sort the strings in lexicographic order
        sorted_strs = [sorted(s) for s in strs]
        
        # create a dictionary to store the frequency of each character in each string
        freq_dict = {}
        for s in sorted_strs:
            for c in s:
                if c not in freq_dict:
                    freq_dict[c] = 0
                freq_dict[c] += 1
        
        # iterate over the characters and add the strings that have the same frequency to a list
        for c, freq in freq_dict.items():
            key = str(freq)
            
            if key not in result:
                result.append([str(i) for i in range(len(strs)) if sorted_strs[i][0] == c])
                
        return result
```

This solution works by first sorting the input strings in lexicographic order. Then, it creates a dictionary `freq_dict` to count the frequency of each character in each string. Finally, it iterates over the characters and adds the strings that have the same frequency to a list.

The time complexity of this solution is O(n * m), where n is the number of strings and m is the maximum length of any string. This is because we need to iterate over all the characters in all the strings, and each iteration takes constant time.

The space complexity of this solution is also O(n * m), since we need to store the frequency of each character in the `freq_dict`. However, since we only use a fixed-size dictionary with at most m keys (where m is the maximum length of any string), the actual space used will be less than or equal to m.
