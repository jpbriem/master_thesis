def longest_common_subsequence(X, Y):
    m = len(X)
    n = len(Y)

    dp = [[0 for _ in range(n+1)] for _ in range(m+1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):

            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1

            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    lcs_length = dp[m][n]
    lcs = []

    i = m
    j = n

    while i > 0 and j > 0:

        if X[i-1] == Y[j-1]:
            lcs.append(X[i-1])
            i -= 1
            j -= 1

        elif dp[i-1][j] >= dp[i][j-1]:
            i -= 1

        else:
            j -= 1

    return ''.join(reversed(lcs))

This is a dynamic programming approach that builds up the length of the longest common subsequence (LCS) between two strings `X` and `Y`. The algorithm works by creating a table `dp`, where each cell `(i,j)` represents the length of the LCS between the first `i` characters of `X` and the first `j` characters of `Y`.

The algorithm starts at the bottom-right corner of the table and iterates through each row and column, filling in the values based on whether the current characters match or not. If they do match, the value in the current cell is set to one more than the value in the cell diagonally above and to the left. Otherwise, the value in the current cell is set to the maximum of the value in the cell above and the value in the cell to the left.

Once the table is filled in, the algorithm can use it to backtrack and construct the LCS itself. Starting from the bottom-right corner, the algorithm checks if the current character of `X` matches the current character of `Y`. If it does, the algorithm adds the current character to the LCS and moves diagonally up-left to the previous cell. If it doesn't, the algorithm moves either upwards or to the left depending on which has a higher value in the current cell.

Finally, the algorithm returns the reversed string representation of the LCS.
