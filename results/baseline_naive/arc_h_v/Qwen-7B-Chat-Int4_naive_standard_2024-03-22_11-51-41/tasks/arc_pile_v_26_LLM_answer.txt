To solve this problem, we can use a hash table to keep track of the characters that have already been seen in each row. We iterate over each row and for each character, if it has not been seen before, we add it to the corresponding row in the output array. If it has been seen before, we skip it.

Here is the Python code to implement this approach:

```python
def permute_rows(input):
    rows = [list(row) for row in input]
    n = len(rows[0])
    out = [[None] * n for _ in range(len(rows))]

    def backtrack(i, j):
        if i == len(rows):
            out[j][0:j+1] = rows[i-1]
            return

        for k in range(n):
            if out[j][k] is None or out[j][k] != rows[i-1][k]:
                out[j][k] = rows[i-1][k]
                backtrack(i, j + 1)

    backtrack(0, 0)
    return out
```

The time complexity of this solution is O(n^2), where n is the length of the longest row in the input. This is because we need to check all possible permutations of characters in each row, which takes O(n!) time. The space complexity is also O(n^2), since we need to store the output array.
