```python
class Solution:
    def matrixBlockSum(self, grid: List[List[int]]) -> List[List[int]]:
        if len(grid) == 0 or len(grid[0]) == 0:
            return []
        
        m = max(len(grid), len(grid[0]))
        n = min(len(grid), len(grid[0]))
        
        block_size = int(m/n)
        
        result = [[0 for _ in range(n)] for _ in range(m)]
        
        for i in range(0, m - block_size + 1, block_size):
            for j in range(0, n - block_size + 1, block_size):
                temp_sum = sum(grid[i:i+block_size][j:j+block_size])
                
                for k in range(i, i+block_size):
                    for l in range(j, j+block_size):
                        result[k][l] += temp_sum
                    
        return result
```

Explanation:

The problem can be solved using dynamic programming. We will create a new list of lists `result` with the same dimensions as the input grid.

We iterate over all possible blocks of size `block_size` (the minimum number of rows and columns required to cover the entire grid). For each block, we calculate its sum by summing up all the elements in that block. Then, we add this sum to the corresponding element in the `result` list at the same position.

Finally, we return the `result` list which contains the sums of all blocks of the input grid.

Time complexity: O(m * n * b), where `m` is the number of rows, `n` is the number of columns, and `b` is the block size.
Space complexity: O(m * n), since we are creating a new list of lists to store the results.
