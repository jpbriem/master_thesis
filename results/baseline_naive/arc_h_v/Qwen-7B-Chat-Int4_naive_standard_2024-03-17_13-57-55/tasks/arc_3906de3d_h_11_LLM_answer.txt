```python
class Solution:
    def maxChunksToSorted(self, arr: List[int]) -> int:
        n = len(arr)
        
        if n == 0 or n == 1:
            return n
        
        i = j = 0
        while i < n and j < n:
            # Move all elements less than the pivot to left side of array
            while i < n and arr[i] <= arr[j]:
                i += 1
            
            # Move all elements greater than the pivot to right side of array
            while j < n and arr[j] > arr[i]:
                j += 1
                
            # If we have moved an element from right side of array,
            # then move it back to left side
            if j - i > 0:
                arr[j-1], arr[i] = arr[i], arr[j-1]
            
        # Now we know that elements on left are sorted in ascending order.
        # Elements on right are also sorted in ascending order. 
        # We can use this fact to sort the entire array efficiently.
        
        # Copying the first half of elements into a new array
        mid = (n + 1) // 2
        result = []
        for k in range(mid):
            result.append(arr[k])
            
        # Sorting the copied array
        result.sort()
        
        # Copying remaining half of elements into a new array
        for k in range(mid, n):
            arr[k] = result.pop(0)
            
        return n
```

Explanation:

The solution uses two pointers `i` and `j`. Pointer `i` is used to traverse the input array from left to right, while pointer `j` is used to traverse the input array from right to left. The idea is to move all elements less than the pivot to the left side of the array, and all elements greater than the pivot to the right side of the array.

Once the pointers meet, we can swap the last two elements in the array so that they become adjacent. This effectively moves the pivot element to its correct position in the array.

We repeat this process until all elements are sorted. At this point, we copy the first half of elements into a new array, sort them, and copy the remaining half of elements into the original array.

This approach has a time complexity of O(n log n), where n is the length of the input array. It works by sorting only the elements that need to be sorted, which reduces the number of comparisons required compared to sorting the entire array using traditional algorithms like bubble sort or merge sort.
