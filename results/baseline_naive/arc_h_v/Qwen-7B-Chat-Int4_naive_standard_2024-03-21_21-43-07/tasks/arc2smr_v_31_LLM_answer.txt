This is a problem of finding the shortest path in a grid using dynamic programming. The approach to solve this problem is to use a 2D array `dp` where `dp[i][j]` represents the length of the shortest path from cell (0, 0) to cell (i, j). We can fill in the values of `dp` by considering all possible moves from each cell and updating `dp[i][j]` based on the minimum distance found so far.
Here's the Python code that implements this approach:
```python
def minDistance(grid):
    m = len(grid)
    n = len(grid[0])
    
    dp = [[float('inf')] * n for _ in range(m)]
    dp[0][0] = 0
    
    for i in range(1, m):
        for j in range(1, n):
            if grid[i-1][j-1] == 'e':
                dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + 1
            else:
                dp[i][j] = min(dp[i][j-1], dp[i-1][j]) + 1
    
    return dp[m-1][n-1]
```

The time complexity of this algorithm is O(m*n), where m and n are the number of rows and columns in the grid respectively. This is because we need to iterate over every cell in the grid once to compute the distances. The space complexity is also O(m*n), as we need to store the distance values for every cell in the grid.
