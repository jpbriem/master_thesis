```python
def longest_common_subsequence(nums):
    if not nums or len(nums) == 0:
        return []
    
    m = len(nums)
    n = len(nums[0])
    
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m+1):
        for j in range(1, n+1):
            if nums[i-1] == nums[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    
    result = []
    i, j = m, n
    
    while i > 0 and j > 0:
        if nums[i-1] == nums[j-1]:
            result.append(nums[i-1])
            i -= 1
            j -= 1
        elif dp[i-1][j] >= dp[i][j-1]:
            i -= 1
        else:
            j -= 1
    
    return result[::-1]
```

This implementation uses dynamic programming to build a table `dp` where `dp[i][j]` represents the length of the longest common subsequence between the first `i` elements of `nums` and the first `j` elements of `nums`. The algorithm then iterates over this table to construct the final result.

For example, given the input `[7, 7, 7, 7, 0, 0, 8, 0, 0]`, the output would be `[0, 0, 0, 1, 1, 1, 1, 8, 0]`. This can be seen by observing that the only non-zero entries in the `dp` table are those corresponding to pairs of adjacent ones. These correspond to the locations where two numbers are equal, so the longest common subsequence must contain these two numbers. All other pairs have a shorter common subsequence, which is constructed from the previous pair.
