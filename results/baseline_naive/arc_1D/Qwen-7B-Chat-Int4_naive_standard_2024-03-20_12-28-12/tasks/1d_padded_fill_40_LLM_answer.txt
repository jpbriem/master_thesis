```python
class Solution:
    def longestPalindrome(self, s: str) -> int:
        if len(s) < 2:
            return 0

        start = end = max_len = 0
        for i in range(len(s)):
            if i <= end and self.is_palindrome(s[start:end+1]):
                end += 1
            else:
                start = i + 1
                end = i - 1

            max_len = max(max_len, end - start + 1)

        return max_len

    def is_palindrome(self, s):
        return s == s[::-1]
```

In each iteration of the outer loop, we check all possible starting indices of palindromes with lengths greater than or equal to the current maximum length (`max_len`). We do this by checking whether the substring from index `start` to index `end-1` is a palindrome using the `is_palindrome` function.

If it is a palindrome, we update the maximum length and move the end index forward to `end`. If it is not a palindrome, we reset the start index to `i+1` (the next character after the non-palindrome), and move the end index back to `i-1` (the previous character before the non-palindrome).

At the end of each iteration, we return the maximum length found so far.

Note that the time complexity of this solution is O(n^2), where n is the length of the input string `s`. This is because we need to check all possible substrings of `s` with lengths greater than or equal to the current maximum length. However, since the maximum length is updated in each iteration, the actual number of iterations required is less than n^2.
