To solve this problem, we can iterate over the input list and keep track of two sets: one set contains all unique characters in the current iteration, and another set contains all unique characters that have appeared before. We can then check if a character has already been seen by checking if it is in either set. If it hasn't been seen before, we add it to both sets.
Here's an implementation in Python:

```python
def group_unique_chars(input):
    output = []
    seen_chars = set()
    for char in input:
        if char not in seen_chars:
            seen_chars.add(char)
            output.append(char)
    return output
```

This solution has a time complexity of O(n), where n is the length of the input list, since we only need to iterate over the list once. The space complexity is also O(n) due to the need to store the set of seen characters.
