To solve this problem, we can use a sliding window approach. We will maintain two pointers, one at the beginning of the array and one at the end. The pointer at the beginning will always move forward, while the pointer at the end will move both forward and backward.
At each step, if the character at the end pointer is equal to the character at the beginning pointer, we remove both characters from the array and increment the pointers. Otherwise, we add the character at the end pointer to the output array and move the end pointer forward.
Here's the Python code that implements this algorithm:

```python
def find_duplicate_chars(input):
    # Initialize the pointers
    left = 0
    right = len(input) - 1

    # Create an empty list for storing duplicate characters
    duplicates = []

    # Iterate until the pointers meet in the middle
    while left < right:
        # Get the current character at the beginning pointer
        char_at_left = input[left]

        # Check if there are any characters after the current character
        # If not, we need to check if the next character is also the same as the current character
        if right == left + 1:
            # Move the right pointer forward and check if the next character is the same
            char_at_right = input[right]
            if char_at_right == char_at_left:
                duplicates.append(char_at_left)
                left += 1
                right -= 1
            else:
                right += 1
        else:
            # Add the current character to the output array
            duplicates.append(char_at_left)

            # Move the left pointer forward
            left += 1

    # Return the duplicate characters
    return duplicates
```

This solution has a time complexity of O(n), where n is the length of the input array. This is because we only iterate through the array once, and each iteration takes constant time. The space complexity is also O(1), since we only store a small number of variables.
