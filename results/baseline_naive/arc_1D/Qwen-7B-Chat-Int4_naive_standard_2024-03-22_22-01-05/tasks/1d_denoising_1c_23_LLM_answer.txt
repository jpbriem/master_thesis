To solve this problem, we can use a sliding window approach. We maintain two pointers `i` and `j`, where `i` is the starting index of the current window and `j` is its ending index.

At each step, we check if the character at position `j` is equal to `d`. If it is not, then we append it to the output array and move both pointers forward by one. Otherwise, we remove all characters from the input array that come before position `i`, and move only the character at position `i` to the output array. This ensures that we always have at least one `a` in the window.

Here's the Python code for the solution:

```python
def longest_consecutive(nums):
    i = j = 0
    ans = []
    while i < len(nums) and j < len(nums):
        if nums[j] == "d":
            j += 1
        else:
            while i < j and nums[i] != "a":
                i += 1
            if i < j:
                ans.append(nums[i])
            i += 1
    return "".join(ans)
```

The time complexity of this algorithm is O(n), where n is the length of the input array. The space complexity is also O(n), as we need to store the output string in memory.
