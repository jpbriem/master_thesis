## Answer (4)

You can use a sliding window approach. The idea is to keep track of the last time you saw a non-zero value and then check if that was within some threshold distance from your current position. If it was, then we know there's been no zeroes in between so we can copy over the previous non-zero value. Otherwise, we just set our new value as 0. Here's how this would look like implemented in Python:

```
def fill(nums):
    # Keep track of the last time we saw a non-zero number
    prev = -len(nums)
    ans = []
    for i, num in enumerate(nums):
        # Check if the last non-zero number was within k steps away
        if num!= 0 and i - prev > k:
            # Copy over the previous non-zero number
            ans += [num] * (i - prev)
        elif num == 0:
            # We don't need to do anything here since we already have a 0
            pass
        else:
            # Update the last seen index
            prev = i
    return ans + [0] * (k + 1 - len(ans))
```

This runs in \$O(n)\$ time where \$n\$ is the length of the input list. This works because each element gets processed exactly once. It also uses \$O(n)\$ space due to storing all the elements in the answer list.
